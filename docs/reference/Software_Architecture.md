## 3. Autoload 模块详解

Autoload 是 Godot 中的全局单例，在游戏启动时自动加载，所有场景都可以直接访问。

### 3.1 EventBus (事件总线)

```
┌─────────────────────────────────────────────────────────────────┐
│                         EVENT BUS                                │
│                    事件总线 - 模块解耦通信                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐           │
│   │ GameState   │   │ AudioManager│   │StoryManager │           │
│   └──────┬──────┘   └──────┬──────┘   └──────┬──────┘           │
│          │                 │                 │                  │
│          │    emit_signal()│                 │                  │
│          └────────────────┼─────────────────┘                  │
│                          │                                     │
│                   ┌──────▼──────┐                             │
│                   │  EventBus   │                             │
│                   │  Class      │                             │
│                   └──────┬──────┘                             │
│                          │ emit_signal()                       │
│          ┌───────────────┼───────────────┐                     │
│          │               │               │                     │
│          ▼               ▼               ▼                     │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│   │ClueManager  │  │ DialogueScene│  │ExploreScene│            │
│   └─────────────┘  └─────────────┘  └─────────────┘            │
│                                                                  │
├─────────────────────────────────────────────────────────────────┤
│  Signals:                                                        │
│  ├── game_state_changed(from, to)                               │
│  ├── story_started(story_id)                                     │
│  ├── story_progressed(chapter)                                  │
│  ├── clue_collected(clue_id)                                    │
│  ├── dialogue_started(npc_id)                                    │
│  ├── ui_show_toast(message)                                     │
│  └── voice_input_started/finished/recognized                    │
└─────────────────────────────────────────────────────────────────┘
```

**职责**: 解耦各模块间的通信，所有模块通过事件总线发送和接收通知。

## 8. 关键设计决策

### 8.1 为什么使用 EventBus？

- **解耦**: 各模块不需要直接引用对方，减少耦合
- **可扩展**: 新增功能只需订阅/发布事件，无需修改现有代码
- **可测试**: 可以单独测试各模块，通过事件交互

### 8.2 状态机模式

- GameState 使用状态模式管理场景转换
- 明确的转换规则，防止非法状态跳转
- 状态历史记录，支持返回上一场景

### 8.3 AI服务设计

- 支持云端API（OpenAI GPT-4）和本地备选
- 请求失败时自动降级到模板故事
- 缓存机制减少API调用
